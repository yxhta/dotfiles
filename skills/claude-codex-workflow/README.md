# claude-codex-workflow

Claude Code が実装を直接担当し、レビューだけを Codex に委ねる品質ループのスキル。
実装者とレビュワーを別エージェントに保つことが独立性(=品質担保)の仕組み。

- **Claude(実装)**: このスキルを使うエージェント自身。ファイルを直接編集し、検証・修正も自分で行う
- **Codex(レビュー)**: companion script のネイティブレビュー。編集は禁止、唯一の独立したレビュー役

## 前提条件

- Claude Code に `codex` プラグイン(openai-codex)がインストール済みであること
- Codex CLI が認証済みであること(未認証なら `/codex:setup` を実行)
- git リポジトリ内で使うこと

## 使い方

Claude Code のセッション内で、次のように依頼するとスキルが発動する:

```text
この機能を実装して、Codexでレビューまで回して
```

```text
codex レビューループで <タスク内容> をやって
```

依頼時に指定できること(省略時は Claude がリポジトリの情報から自律的に導出し、最終レポートで開示する):

| 項目                           | デフォルト                 |
| ------------------------------ | -------------------------- |
| 受け入れ基準                   | 依頼内容から Claude が導出 |
| 検証コマンド(テスト・ビルド等) | リポジトリの慣習から推測   |
| 最大修正サイクル数             | 3                          |

## 動作の流れ

1. Claude が目標・受け入れ基準・検証コマンドを確定し(未指定分はリポジトリから導出して冒頭に開示)、git のベースラインを記録
2. Claude が自分のツール(Edit / Write / Bash)で実装
3. Claude が `git diff` とテスト実行で検証
4. Codex ネイティブレビューを実行。指摘があれば Claude が自分で修正して 2〜4 をループ
5. 通常レビューがクリーンになったら `adversarial-review`(より厳しいレビュー)を最終ゲートとして実行。指摘があれば同様に修正してこのゲートを再実行
6. adversarial レビューも通ったら Claude が最終コンプライアンスチェック(受け入れ基準・CLAUDE.md・スコープクリープ・テストカバレッジ)。バグ探しの独立性は既に Codex のレビューで担保済みなので、ここは基準適合の確認が目的
7. `status` / `changed` / `verified` / `review` / `decisions`(自律判断の記録)/ `risks` の最終レポートを出力

途中でユーザーに確認するのは、破壊的操作・プロダクト意図の判断・回避不能なブロッカーの 3 つだけ。それ以外の判断は Claude が行い、`decisions` に記録される。

## トラブルシューティング

- **プラグインが見つからない**: `~/.claude/plugins/cache/openai-codex/` が存在するか確認し、なければ codex プラグインをインストールする
- **Codex が未認証**: `/codex:setup` を実行する
- **レビュー役の Codex がファイルを編集した**: ユーザーの未コミット変更と重ならないパスは Claude がベースラインに戻してループを続行する(最終レポートに記録)。重なる場合のみ停止して指示を仰ぐ
- **未コミットの変更がある状態で開始した**: 停止せず degraded mode(ベースライン差分でスコープ切り分け)で続行し、最終レポートで開示する。タスク対象ファイルと大きく重なる場合のみ確認が入る
- **最大サイクル到達後も指摘が残る**: 収束傾向が明らかな場合のみ 1 サイクルだけ自動延長する(レポートに記録)。それでも残った指摘は `risks` に載る。継続する場合は再度依頼する

## 補足

- `/codex:review` コマンド自体は `disable-model-invocation: true` のため Claude から呼べない。このスキルはプラグインの companion script(`codex-companion.mjs review`)を直接実行することでレビューを自動化している
- レビューのスコープは自動解決(working tree が dirty なら未コミット差分、クリーンならデフォルトブランチとのブランチ差分)。修正サイクル中は通常の `review`、クリーン後の最終ゲートに `adversarial-review` を使う。急ぎのときは依頼時に「クイックで」と伝えれば adversarial ゲートをスキップできる
- 実装を Codex に委譲したい場合は、このスキルではなく `codex:codex-rescue` サブエージェントを直接使う
- このスキルは codex プラグインと Agent tool に依存する Claude Code 専用。Codex 側から読み込む想定はないため `agents/openai.yaml` は置いていない
